The Font Problem

Link to Article here.Here

One thing one should be careful of when designing a web page is the font they use and I don’t mean style or how it looks. What we are talking about is FOIT (The flash of invisible text), basically when the font is loading for the text, the web page hides it from view. Which brings us to the main point of the article, the “Mitt Romney font problem”.


The problem is when using two different fonts in a header, FOIT may hide part of it from view while it is loading which in this case, caused the article to be misleading. A news website, Slate.com, posted an article on how Mitt Romney was not running for president. However, they used a different font on the ‘Not’ part, so that when the article first loads it says Romney was running for president. Each web font has its own separate FOIT, and when there are two fonts in one headline, it creates a race condition which changes the appearance of the headline initially. So when people go to the article, they think Romney is doing one thing when actually it's the opposite, and in politics this can be dangerous, but it also contributes to an even larger issue.


The spreading of misinformation on the web is not uncommon, gossip and rumors about numerous topics are on the web everywhere. But with this font problem, when news organizations cover something significant and people read the header, get the wrong conclusion before it loads then spread the word to others it can have disastrous consequences. News organizations rely on their articles to be clear and accurate. For example if this had about COVID-19, and that it was not spreading, if this problem occurred, people would read it as it was spreading and it would induce panic among the population. Understanding the repercussions this problem has, what can be done going forward?


There are solutions both developers and browsers can use to address this font problem. For developers, one solution involves having unstyled fallback text as a placeholder while the font is loading. Another involves using font-synthesis to show bold and italic fallback text generated by the browser meanwhile the real web fonts load. And the last involves grouping font loads into a single repaint, having them load together instead of separately. Browsers should also be able to solve this by blocking font-rendering behavior in a more suitable way, assigning a way for fonts to be prioritized in a smarter fashion.
