Modules

JavaScript Module's are clusters of code that are self-contained and have specific functions that they serve. Their basic purpose is to make code more easy to maintain and update without effecting the rest of the application.

Module Pattern

A Module pattern is one of the various ways to implement modules in your code. It works in a similar way to how classes work in other languages, since JavaScript classes aren't supported natively. This way one can store private and public functions and variables in a single object. Here are some examples of Module Patterns.

Module Bundling

Now one might ask, what is module bundling, simply put its a way of grouping together modules into and single or multiple files. In an HTML file, one has to attach JavaScript through the script tag, which would slow down the loading times for pages. So combining modules into single or few files is done to avoid this. When using module patterns, to bundle them you will want to concatenate them into a single file and/or minify the code by removing unnecessary characters from the bundled code to reduce space.

Bundling code for ES6 modules is a little more complex. For example using a transpiler, Babel for example, that converts ES6 code to CommonJS, AMD, or UMD format (these are other methods of incorporating modules.)
