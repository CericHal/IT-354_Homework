Node and V8

Link to Article here.

Node.js is an open source JavaScript runtime environment that is built upon the V8 JavaScript. Node.js was launched in 2009 with the purpose of allowing developers to create more server-side applications through JavaScript by running code through the computer process instead of a web browser.


V8 is an open source JavaScript engine that powers Node.js and since then has been an integral part of web development. It takes in JavaScript code, but as it's running it translates JavaScript into machine code allowing for very fast execution.Today it is used in Google Chrome and provides the environment where our JavaScript code executes in the browser. Other browsers use different JS engines, but in this article we are focusing on the V8 engine specifically. V8 was coded in C++, and since its release it has been constantly updated and is available for multiple operating systems including Windows, Mac, and Linux. So now that you know what the V8 engine is, here is a brief look at how it came to be.


So now that we know what the V8 engine is, let's take a look at a brief history of the JS engine. It was developed by the Chromium Project for google Chrome. It was released back in September 2, 2008 along with the first version of Chrome and was based on the Strongtalk assembler. Fast forward a couple years, a new compiler infrastructure was introduced to V8 called Crankshaft, which improved speed for compiling code. Come 2016, the ignition interpreter was added to V8, aiming to reduce memory use on Android phones with small memory. In 2017 V8’s compiler infrastructure was drastically changed in version 5.9 where V8’s compilers, Full-codegen and Crankshaft no longer compiled JavaScript in V8 since they couldn’t keep pace with technical advancements. Today V8 is currently on version 8.1.


Now you might be wondering how Node.js and the V8 JS engine are related to each other. See V8 engine can be run outside of the browser through Node.js, allowing the user to write and run JavaScript code without having to rely on the browser. Basically Node.js provides an environment for developers to display their code’s input and output, and V8 is the one doing the compiling the JavaScript code.


Personally, as a developer I always assumed JavaScript was capable of server-side development and it never occured to me that Node.js helped change how JavaScript as a language is used today. JavaScript is a single thread language, meaning only one thing can happen at a time, yet Node.js is built to handle asynchronous code to accomplish more in less time. Node uses an event driven model, so that way it allows the code to flow easier and faster, and it reduces the input/output backup in the code.In addition to that, the fact that V8 was designed to run JavaScript in chrome yet is also independent from the browser it is run on is also fascinating. These are just a few things I find fascinating about Node.js and the V8 engine and hopefully you found something interesting here too.
