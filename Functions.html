ECMAScript: A history

ECMAScript is the scripting language that forms the basis of JavaScript. See JavaScript was developed for the browser Netscape 2 in 1996 by Brenden Eich, considered to be the father of JavaScript, who came into the picture when he was contracted by NetScape communications to create the first version of JavaScript. This is where ECMA comes in, ECMA is an industry association whose purpose is to standardize information and communication systems. ECMA began work on standardizing JavaScript in November of 1996, not even a year since the programming language came out. What was produced as a result is ECMAScript, which today is commonly known today as JavaScript. ECMAScript 1 and 2 still missed a lot of important features like JSON, exceptions and methods for builtin methods. Version 1 was released in June of 1997. Version 2 was released in June of 1998 and was mainly editorial changes. Version 3 had the first big changes, including regular expressions, the do-while block, the try/catch blocks, the in and instance of operators, and built-in functions for strings and arrays. This version was released in December of 1999. ECMAScript 4 was never released mainly because the committee couldn’t agree on whether or not to add much more features to the standard. In peak development, some of the features were classes, interfaces, and much, much more. ECMAScript 5, released in 2009, added JSON support, new array methods, and object methods. It was considered to be a rather modest update compared to version 3. Version 6, otherwise referred to as ECMAScript 2015, added the let and const, default parameter values, Array.find(), and Array.findIndex(). This version aimed to make JavaScript more appealing to a larger audience, so it could be used by small startups to industry giants alike. ECMAScript 7 (2016) contained mainly small revisions to the previous version and adding things like the exponentiation operator and Array.prototype.includes. It was the result of a new release process by the developers, where new proposals must go through a 4 stage process. So when enough proposals reach stage 4, a new version can be released and if they don’t reach stage 4, it doesn’t hinder development at all. Helping to prevent what happened to version 4. Since then, two new versions have been released, with the most recent being ECMAScript version 9 as of 2019.
