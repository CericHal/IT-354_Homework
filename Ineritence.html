How 'this' acts in scope of inheritance

The keyword 'this' also acts in a unique way when it comes to inheritance. Notice how in the first picture function luke inherits the attribute values of function anakin and couldn't change them? That was because of the this keyword.

However, in the second picture the inheritor object, can change the value. That's because 'this' acts within a certain scope even for inheritance. When the inherited function was referenced, 'this' to the inheritor object, NOT the original object. So if you want the inheritor object to have immutable values of the parent object use the this keyword.

This keyword

We have seen the 'this' keyword in action with previous examples of code, but what exactly does it do? The 'this' keyword refers to the object that they belong to so when we type this followed by a period and a property name, it refers to the object that the property belongs to. When not in the scope of a class or function, this refers to the global object.


The global object basically refers to the script or file itself. The window object is basically the global object but it is created when running a script in a web browser. All global functions and variables become part of the window object. Try not to use these objects when you can, since the values exist for the lifetime of the application. Meaning they will take up unnecessary space in your application. So keep that in mind when working with the window object.
